## Directory Purposes

**`include/`:**
- Purpose: All C++ public headers.
- Contains: Four headers — three are thin declaration files paired with a `src/*.cpp`, and `cache.h` is an exception where all five cache-policy classes are defined inline.
- Key files: `cache.h`, `trace_gen.h`, `workload_stats.h`, `shards.h`.

**`src/`:**
- Purpose: C++ implementation files compiled into `cache_sim`.
- Contains: Exactly four `.cpp` files — one driver plus three implementation units. No nested subdirectories.
- Key files: `src/main.cpp`, `src/trace_gen.cpp`, `src/workload_stats.cpp`, `src/shards.cpp`.

**`scripts/`:**
- Purpose: Python tooling that sits on either side of the C++ simulator in the data pipeline.
- Contains: Two standalone scripts. Both have a `main()` and are runnable directly.
- Key files: `scripts/collect_trace.py`, `scripts/plot_results.py`.

**`traces/`:**
- Purpose: Raw cache-access traces (CSV with header `timestamp,key,size`).
- Contains: `congress_trace.csv` (large collected trace), `test_trace.csv` (small fixture), and `collect.log` from the most recent collection run.
- Generated: Yes (by `scripts/collect_trace.py`).
- Committed: No (`.gitignore:28`).

**`results/`:**
- Purpose: Simulator CSV outputs and plot PDFs. Each run overwrites its CSVs in-place (Makefile target `run` calls `mkdir -p results`).
- Contains: One CSV per analysis (`mrc.csv`, `alpha_sensitivity.csv`, `one_hit_wonder.csv`, `shards_mrc.csv`, optionally `exact_mrc.csv` and `shards_error.csv`), plus `figures/*.pdf`. A `validation/` subdirectory holds a parallel set of outputs from a smaller exact-SHARDS validation run.
- Generated: Yes (by `cache_sim` and `scripts/plot_results.py`).
- Committed: No (`.gitignore:31`).

**`build/`:**
- Purpose: Out-of-source compilation artifacts.
- Contains: `main.o`, `trace_gen.o`, `workload_stats.o`, `shards.o` with matching `.d` dependency files generated by `-MMD -MP`.
- Generated: Yes (by `make`).
- Committed: No (`.gitignore:2`).

**`.planning/codebase/`:**
- Purpose: GSD codebase-map documents produced by `/gsd-map-codebase`.
- Contains: ARCHITECTURE.md, STRUCTURE.md (this file), and whatever other focus areas have been mapped.
- Generated: Yes (by GSD agents).

## Key File Locations

**Entry Points:**
- `src/main.cpp:77` — `main()` for the `cache_sim` binary.
- `scripts/plot_results.py:296` — plot driver.
- `scripts/collect_trace.py:177` — trace collector.

**Configuration:**
- `Makefile` — compile flags (`-std=c++17 -O2 -Wall -Wextra -Iinclude -MMD -MP`) and `run` / `run-sweep` / `plots` targets.
- `requirements.txt` — Python dependencies.
- `.env` — holds `CONGRESS_API_KEY` (never read by tooling directly; loaded from environment by `scripts/collect_trace.py:51`).

**Core Logic:**
- `include/cache.h` — abstract `CachePolicy` base plus all five concrete eviction policies (inline in header).
- `src/main.cpp` — orchestration, CLI parsing, CSV writing.
- `src/shards.cpp` — SHARDS streaming estimator and exact-MRC reference path.
- `src/trace_gen.cpp` — Zipf CDF construction, CSV load, `replay_zipf`.
- `src/workload_stats.cpp` — Zipf α MLE (Newton's method) and OHW ratio.

**Testing:**
- No unit-test framework. `traces/test_trace.csv` acts as a small end-to-end fixture that can be fed to `./cache_sim --trace traces/test_trace.csv` for quick validation.
- `results/validation/` contains reference outputs from a prior `--shards-exact` run for comparing approximate MRC against exact MRC.

**Documentation:**
- `README.md` — public-facing usage, build instructions, policy list, references.
- `PROCESS.md`, `progress.tex`, `progress.pdf`, `v7.tex` — internal working documents (all git-ignored).

## Naming Conventions

**Files:**
- C++ headers: lowercase snake_case with `.h` extension (`trace_gen.h`, `workload_stats.h`).
- C++ sources: matching lowercase snake_case `.cpp` files in `src/` (`trace_gen.cpp`, `workload_stats.cpp`).
- Python scripts: lowercase snake_case with `.py` extension (`collect_trace.py`, `plot_results.py`).
- CSV outputs: lowercase snake_case (`mrc.csv`, `alpha_sensitivity.csv`, `one_hit_wonder.csv`, `shards_mrc.csv`, `exact_mrc.csv`, `shards_error.csv`).

**C++ Classes:**
- `PascalCase` for types and classes (`CachePolicy`, `LRUCache`, `FIFOCache`, `CLOCKCache`, `S3FIFOCache`, `SIEVECache`, `ZipfGenerator`, `SHARDS`, `TraceEntry`, `CacheStats`, `WorkloadStats`, `MRCPoint`).
- Acronyms stay uppercase (`LRU`, `FIFO`, `CLOCK`, `SHARDS`, `MRC`). Multi-word acronyms retain hyphens or camel separators in display strings only (`"S3-FIFO"` via `name()`; class is `S3FIFOCache`).

**C++ Functions / Methods:**
- `snake_case` for free functions and member methods (`load_trace`, `generate_zipf_trace`, `replay_zipf`, `characterize`, `estimate_zipf_alpha`, `one_hit_wonder_ratio`, `exact_stack_distances`, `mrc_from_stack_distances`, `build_mrc`, `process`).
- Private helpers inside `main.cpp` use `static` + snake_case (`print_usage`, `split`, `working_set_bytes`, `make_policy`, `run_simulation`).

**C++ Variables / Members:**
- `snake_case` for locals and parameters.
- Trailing underscore for private data members (`capacity_`, `current_size_`, `order_`, `map_`, `hand_`, `rate_`, `threshold_`, `modulus_`, `last_access_`, `access_times_`, `sd_hist_`, `total_sampled_`, `rng_`, `cdf_`).

**Python:**
- `snake_case` for functions and variables.
- `SCREAMING_SNAKE_CASE` for module-level constants (`BASE_URL`, `ENDPOINTS`, `CURRENT_CONGRESS`, `CURRENT_CONGRESS_WEIGHT`, `POLICY_COLORS`, `POLICY_MARKERS`).

**CLI Flags:**
- Double-dash long form, hyphen-separated (`--trace`, `--cache-sizes`, `--output-dir`, `--num-requests`, `--num-objects`, `--alpha-sweep`, `--shards`, `--shards-exact`, `--replay-zipf`). No short aliases except `-h`.

**Policy String IDs:**
- Lowercase when used as CLI identifiers (`lru`, `fifo`, `clock`, `s3fifo`, `sieve`); canonical display strings returned by `name()` use uppercase with a hyphen for S3-FIFO (`"LRU"`, `"FIFO"`, `"CLOCK"`, `"S3-FIFO"`, `"SIEVE"`).

**CSV Column Names:**
- `snake_case` (`cache_frac`, `cache_size_bytes`, `cache_size_objects`, `miss_ratio`, `byte_miss_ratio`, `sampling_rate`, `window_frac`, `ohw_ratio`, `mae`, `max_abs_error`, `num_points`).

## Where to Add New Code

**New cache-eviction policy:**
- Primary code: append the class inline in `include/cache.h`, inheriting from `CachePolicy`. Follow the `// ==================== NAME ====================` section banner pattern used by the existing five.
- Factory registration: add a branch in `make_policy()` at `src/main.cpp:59-66`.
- Default-policies list: add the lowercase ID to `policy_names` default at `src/main.cpp:82`.
- Display label mapping: add any special uppercase / hyphenated label handling in the two places that render headers (`src/main.cpp:173-178` and `src/main.cpp:210-215`).
- Plot color/marker: extend `POLICY_COLORS` and `POLICY_MARKERS` in `scripts/plot_results.py:31-45`.
- README: add a row to the policies table in `README.md`.

**New workload statistic or analysis pass:**
- Add the function signature to `include/workload_stats.h`, implement in `src/workload_stats.cpp`.
- If it produces its own CSV: add a top-level block in `src/main.cpp` following the pattern of the one-hit-wonder section (`src/main.cpp:241-256`) — open `std::ofstream`, write header, loop, close.
- Add a `plot_<name>` function in `scripts/plot_results.py` and register it in `main()` at `scripts/plot_results.py:307-314`.

**New trace source or transformation:**
- Declare in `include/trace_gen.h`, implement in `src/trace_gen.cpp`.
- Wire into the trace-selection block at `src/main.cpp:129-151` with a new CLI flag if needed.

**New SHARDS variant or MRC estimator:**
- Extend `include/shards.h` / `src/shards.cpp`. For a new sampler, mirror the existing `SHARDS` class interface so the `main.cpp` SHARDS section (`src/main.cpp:259-339`) can reuse `process()` + `build_mrc()` shapes.

**New CLI flag:**
- Add a branch in the `for (int i = 1; i < argc; i++)` loop at `src/main.cpp:92-125`.
- Add the flag to `print_usage()` at `src/main.cpp:18-33`.
- Document it in `README.md` under "CLI options".

**New Python utility:**
- Add a new file under `scripts/` with `#!/usr/bin/env python3` shebang, a module docstring, and an `argparse`-driven `main()`.
- Append any new Python dependencies to `requirements.txt`.

**Generated outputs:**
- Write to `results/` (or the path passed via `--output-dir`). Do not commit the files — `results/` is git-ignored.

**Tests / fixtures:**
- Small deterministic traces go in `traces/` (git-ignored). Generate them via `./cache_sim --num-requests 5000 --num-objects 500` and redirect output, or write them manually with the `timestamp,key,size` schema.

- Purpose: Python virtual environments (two pinned Python versions).

**`.planning/`:**

- Committed: Typically yes for codebase maps (no explicit ignore rule).
