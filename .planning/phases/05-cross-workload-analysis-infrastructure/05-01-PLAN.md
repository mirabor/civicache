<objective>
Add a single `--seed N` CLI flag to `cache_sim` that threads into all Zipf trace generation / replay call sites. This unblocks Plan 05-03 (multi-seed sweep orchestrator) which requires per-seed variability in the Zipf permutation and sampling. All other Phase 5 plans are pure Python scripts operating on the CSVs this seed produces.

**Why this is scoped tight:** D-06 in 05-CONTEXT.md originally said the seed threads into BOTH `generate_zipf_trace` AND the SHARDS sampler. **Corrected by pattern-mapper:** SHARDS uses content-addressed FNV-1a hashing (deterministic — same key always maps to the same hash; no internal RNG to seed). Therefore `src/shards.cpp` is NOT modified. Only `src/main.cpp` argparse + call-site threading.

**Why seed variation still produces meaningful variability across 5 seeds (no SHARDS involvement):** The Zipf replay path re-shuffles the prepared object list via `prepare_objects(raw_trace, seed)` and re-picks the ZipfGenerator stream via `generate_replay_trace(..., seed)` — so different seeds produce different access orderings over the same key universe. The cache's state evolves differently, yielding different miss_ratio values. This is the variability multi-seed CI claims rest on. SHARDS's deterministic sampling is orthogonal and does not need to vary.

**Back-compat contract (D-06):** absent `--seed` reproduces Phase 1-4 results byte-for-byte. The default value `seed = 42` matches the existing header defaults on `generate_zipf_trace`, `replay_zipf`, `prepare_objects`, and `generate_replay_trace` (all `uint64_t seed = 42` per include/trace_gen.h lines 17/26/34/41/49). The `--emit-trace` block at src/main.cpp line 178 continues to call `generate_zipf_trace(num_requests, num_objects, alpha, 42)` with the literal 42 — traces/shards_large.csv provenance requires this.

**Purpose:** ANAL-02 foundations — "Multi-seed runs (5 seeds) for all final policy comparisons with confidence intervals." Without `--seed`, the same trace + alpha + policy always produces the same miss_ratio, and the "±1σ across 5 seeds" story collapses to "a single point".

**Output:** A ~6-line diff to `src/main.cpp` — one variable declaration, one argparse branch, one print_usage line, and four call sites threading the new variable. ~15 total edited lines.

</objective>
