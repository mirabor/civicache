**Environment + session:**

- `get_api_key()`: read `os.environ.get("COURTLISTENER_API_KEY")`; if falsy, print "Error: set COURTLISTENER_API_KEY environment variable (see .env.example; .env is gitignored)" to stderr and `sys.exit(1)`.
- Session setup: `session = requests.Session(); session.headers.update({"Authorization": f"Token {api_key}"}); session.params = {"format": "json"}`.

**Per-request URL construction + host check:**

```python
def build_request(name, endpoints_state):
    spec = endpoints_state[name]
    if "court_ids" in spec:
        cid = random.choice(spec["court_ids"])
        path = spec["path_tpl"].format(court_id=cid)
    else:
        lo, hi = spec["id_range"]
        i = random.randint(lo, hi)
        path = spec["path_tpl"].format(id=i)
    url = BASE_URL + path
    # D-01 hard allowlist — belt-and-suspenders against a malformed path_tpl.
    from urllib.parse import urlparse
    assert urlparse(url).netloc == ALLOWED_HOST, f"Host allowlist violation: {url}"
    params = {}
    if name == "opinion" and random.random() < OPINION_METADATA_FRACTION:
        params["fields"] = OPINION_METADATA_FIELDS  # D-01 / D-02 80% metadata draw
    return path, url, params
```

**Equal-endpoint scheduler (D-05):**

Round-robin through `ENDPOINT_ORDER`, skipping any endpoint that has already reached its remaining target. Track `success_count[ep]` and compute `remaining[ep] = per_endpoint_target[ep] - success_count[ep]`. Stop the main loop when `sum(success_count.values()) >= target_total`.

**Request handling (per iteration):**

1. Pick the next endpoint `ep`; build `(path, url, params)` via `build_request(ep, ...)`.
2. Timestamp `ts = int(time.time() * 1000)`.
3. Try `resp = session.get(url, params=params, timeout=30)`. On `requests.RequestException`, treat as transient: increment `consecutive_failures`, `backoff = min(BASE_DELAY * (2 ** consecutive_failures), 300)`, `time.sleep(backoff)`, continue.
4. Classify `code = resp.status_code` into the STATUS_BUCKETS dict; increment `tally[ep][bucket]`.
5. **D-11 429 ramp:** if `code == 429`: set `retry_after = min(int(resp.headers.get("Retry-After", "0")) or 60, 120)`; increment `consec_429[ep]`; if `consec_429[ep] >= CONSECUTIVE_429_HARD_STOP`, print `HARD_STOP_MSG` to stderr, write the collection report in its current state, `sys.exit(2)`. Otherwise `sleep_s = retry_after + RETRY_AFTER_ADDITIONS[min(consec_429[ep] - 1, 2)]`, `time.sleep(sleep_s)`, continue (DO NOT record the row).
6. Reset `consec_429[ep] = 0` on any non-429 response.
7. If `code == 200` and `len(resp.content) > 0`: `key = path.lstrip("/")` (matches pilot CSV — no leading slash, keeps trailing slash); `writer.writerow([ts, key, len(resp.content)]); f.flush(); success_count[ep] += 1`. D-04: record actual byte length regardless of whether the 20% draw landed on a null-plain_text opinion.
8. On any non-200 (404, 403, 5xx, other): just increment tally and continue; no row written (D-10).
9. Progress print every 100 successful rows; endpoint tally dump every 1000 rows (mirrors `scripts/collect_trace.py` pattern, per D's Claude's Discretion).
10. Jittered sleep: `time.sleep(BASE_DELAY + random.uniform(0, JITTER))`.

**D-06 fallback (first-500-per-endpoint <60%):**

After any endpoint has issued exactly `FALLBACK_PROBE_WINDOW` requests (tally total == 500) and before issuing the 501st, compute `success_rate = tally[ep]["200"] / 500`. If `success_rate < FALLBACK_SUCCESS_THRESHOLD`, narrow that endpoint's `id_range`:
- For ID-range endpoints: `hi_new = int(hi * FALLBACK_NARROW_FACTOR); endpoints_state[ep]["id_range"] = (lo, hi_new)`.
- For the court endpoint: keep as-is (court_ids list is already curated). If court hits the fallback, log the anomaly but don't narrow.
- Zero `success_count[ep]` and `tally[ep]` counters so the new range gets 5K fresh draws.
- Append a line to `collection_report.txt` (via the final-report writer, buffered until end): `f"  [FALLBACK] {ep}: first {FALLBACK_PROBE_WINDOW} at {success_rate:.1%} — narrowed id_range to (1, {hi_new}); counters reset"`.

**--resume flag (D-09):**

If `--resume` is passed:
- Read the existing `traces/court_trace.csv`. Assert it exists and `wc -l >= 2` (header + at least one row). If not, print "Nothing to resume — use normal invocation" and exit 1.
- Parse each row's `key` column; derive endpoint by prefix: `dockets/*` -> docket, `opinions/*` -> opinion, `clusters/*` -> cluster, `courts/*` -> court. Build `success_count[ep]` from these.
- Compute `remaining[ep] = max(0, PER_ENDPOINT_TARGET - success_count[ep])`.
- Open the CSV in append mode (`mode="a"`); do NOT rewrite the header.
- Continue the round-robin loop as normal; skip endpoints whose remaining == 0.

Without `--resume`: if the output CSV exists and is non-empty, print "Error: output CSV exists and is non-empty. Use --resume to continue, or delete/move the file." to stderr and `sys.exit(1)`. Write the CSV header as the first row when opening for a fresh run.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| collector -> www.courtlistener.com | HTTPS egress to single allowlisted host; untrusted response bodies parsed only as opaque byte-length (D-04) |
| .env -> collector process | `COURTLISTENER_API_KEY` flows env-var -> `requests.Session.headers["Authorization"]`; never logged, never written to disk |
| collector -> traces/court_trace.csv | Timestamp + URL-path + response byte-length only; no response body content stored |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering/Info-Disclosure | URL construction (SSRF) | mitigate | Hard allowlist: module-level `assert urlparse(BASE_URL).netloc == ALLOWED_HOST` at import + per-request `assert urlparse(url).netloc == ALLOWED_HOST` in `build_request()`. ID-range and court-ID lists are static Python literals — no user input crosses the boundary. |
| T-03-01-02 | Info-Disclosure | API token leakage | mitigate | `.env` is gitignored (Phase 1 Plan 01-05); only `api_key[:4]+"..."+api_key[-4:]` is ever printed; token lives only in `session.headers["Authorization"]` and env var. No response body is written to disk — only `len(resp.content)`. |
| T-03-01-03 | DoS (self-inflicted) | Rate-limit escalation | mitigate | D-11 429 ramp (Retry-After + [0, 30, 90] additions) and 5-consecutive hard-stop prevent the collector from sustaining a throttle pattern that could escalate to temporary ban. 0.8s + 0-0.4s jitter keeps steady-state ~3200 req/hr, well under CourtListener's 5000/hr authenticated ceiling (STACK.md). |
| T-03-01-04 | Elevation-of-Privilege | Code-injection via API response | accept | No `eval`, no `exec`, no `subprocess` invoked with response data. Response content used only via `len()` and byte-size arithmetic. `requests` library parses HTTP headers safely. |
</threat_model>
