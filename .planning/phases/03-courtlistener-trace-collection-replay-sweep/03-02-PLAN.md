    **Step 4a — Amend `.gitignore` with explicit exemptions for the two files D-15 commits.**

    This is **required** because `traces/` and `results/` are gitignored (lines 28 and 31 of `.gitignore`). A bare `git add traces/court_trace.csv` silently no-ops against an ignored path — the file would not be staged, and D-15's intent (commit the trace) would silently fail. We add `!`-prefixed whitelist entries for the two specific files this plan commits; the `traces/` and `results/` directory ignores remain in place for everything else.

    ```bash
    # Idempotent: only append if the exemptions aren't already present.
    if ! grep -qE '^!traces/court_trace\.csv$' .gitignore; then
        cat >> .gitignore <<'EOF'

# Phase 3 (D-15): committed trace + collection report
!traces/court_trace.csv
!results/court/collection_report.txt
EOF
    fi

    # Verify the exemptions landed.
    grep -qE '^!traces/court_trace\.csv$' .gitignore || { echo "FATAL: .gitignore exemption missing for traces/court_trace.csv"; exit 1; }
    grep -qE '^!results/court/collection_report\.txt$' .gitignore || { echo "FATAL: .gitignore exemption missing for results/court/collection_report.txt"; exit 1; }

    # Confirm git now sees the files as trackable (no longer ignored).
    ! git check-ignore -q traces/court_trace.csv || { echo "FATAL: traces/court_trace.csv is still git-ignored after exemption"; exit 1; }
    ! git check-ignore -q results/court/collection_report.txt || { echo "FATAL: results/court/collection_report.txt is still git-ignored after exemption"; exit 1; }
    ```

    **Step 4b — Commit (D-15).**

    Once all Step 3 checks pass AND Step 4a's `.gitignore` edit is in place, the user runs:

    ```bash
    git add .gitignore
    git add traces/court_trace.csv
    git add results/court/collection_report.txt
    git commit -m "feat(03): collect 20K CourtListener trace + report (TRACE-06)"
    ```
