<objective>
Axis C (Phase 4 D-19): S3-FIFO small-queue-ratio parameter-sensitivity ablation. The Yang et al. (SOSP'23) paper recommends 10% as the default small-queue size; this ablation verifies that recommendation on our two workloads by sweeping {5%, 10%, 20%} and producing a figure showing miss-ratio vs Zipf alpha for each variant on both Congress and Court traces.

Three coordinated changes ship in this plan:
1. **`include/cache.h::S3FIFOCache` ctor extension (D-11):** Add a default-valued `double small_frac = 0.1` parameter and a private `small_frac_` + `name_` member. The existing `capacity / 10` integer-truncation formula generalizes to `static_cast<uint64_t>(capacity * small_frac)` clamped to ≥1. The `name()` method returns the `name_` field, which the ctor sets based on the small_frac value: 0.05 → "S3-FIFO-5", 0.10 → "S3-FIFO-10", 0.20 → "S3-FIFO-20", everything else → "S3-FIFO" (the legacy default name, preserved for back-compat when callers pass no small_frac OR pass exactly 0.10). See PATTERNS.md "include/cache.h — S3FIFOCache ctor" for the legacy-alias subtlety.

2. **`src/main.cpp` factory + label-map extensions:** Three new make_policy branches construct S3FIFOCache with the corresponding small_frac. The legacy `s3fifo` branch stays as `S3FIFOCache(capacity)` (no small_frac arg → ctor default 0.1 with `name_ = "S3-FIFO"` preserved). The two policy-label maps at lines 184-188 and 234-238 gain three new display-name overrides so that table headers print "S3-FIFO-5" etc.

3. **Makefile + plot_results.py extensions:** The `ablation-s3fifo` Makefile target runs the simulator's existing `--alpha-sweep` path twice (once per workload) with `--policies s3fifo-5,s3fifo-10,s3fifo-20`, then renames the produced `alpha_sensitivity.csv` to `ablation_s3fifo.csv` (the alpha-sweep emitter is the natural producer; D-13 fixes cache at 1% via the existing `wb/100` at main.cpp:255). The new `plot_ablation_s3fifo` function in plot_results.py reads both per-workload CSVs and produces a 2-panel figure (Congress | Court) sharing y-axis, with three lines per panel (one per small_frac variant). POLICY_COLORS/POLICY_MARKERS gain the three new dict entries.

**Why this is its own plan:** Axes A/B/D have no overlap with cache.h S3FIFOCache or with `s3fifo*` policy strings. The S3-FIFO ctor change is mechanical and contained; the Makefile target and plot function are independent of the other axes' targets/functions. Per CONTEXT D-19 axis decomposition, this stands as Wave 1 alongside Plans 04-01/04-02/04-04 with zero file conflicts.

**Crucial back-compat constraint (D-11 + Phase 4 L-8 equivalent):** A re-run of `make run-sweep` (Congress) and `make run-sweep WORKLOAD=court TRACE=traces/court_trace.csv` after this plan must produce CSVs whose `S3-FIFO` rows are bit-identical to their pre-Phase-4 counterparts. The integer truncation `capacity / 10` is NOT necessarily equal to `static_cast<uint64_t>(capacity * 0.10)` for every capacity (FP rounding for 0.1 — `0.1 * 100 = 9.999...8` truncates to 9, not 10). To guarantee bit-identity, the legacy `s3fifo` ctor branch must use the EXACT old formula. See Task 1 for the 1-line guard: when `small_frac == 0.1` AND the constructor was reached via the no-arg overload (which is impossible to detect at runtime since both paths set small_frac_=0.1), use `capacity / 10` directly. The cleanest implementation: replace the formula in the unified ctor with `small_capacity_ = std::max<uint64_t>(1, (small_frac == 0.1) ? capacity / 10 : static_cast<uint64_t>(capacity * small_frac))`. This preserves bit-identity for the default path while allowing 0.05 / 0.20 to use the FP formula.

```python
def plot_ablation_s3fifo(figures_dir, congress_dir="results/congress",
                          court_dir="results/court"):
    """S3-FIFO small-queue-ratio ablation (D-11, ABLA-01). 2-workload panel grid.
    Reads results/{congress,court}/ablation_s3fifo.csv, plots miss_ratio vs alpha
    for each of the 3 small_frac variants on each workload at fixed 1% cache (D-13).
    """
    cong_path = os.path.join(congress_dir, "ablation_s3fifo.csv")
    court_path = os.path.join(court_dir, "ablation_s3fifo.csv")
    if not (os.path.exists(cong_path) and os.path.exists(court_path)):
        print(f"  Skipping S3-FIFO ablation plot: both {cong_path} and {court_path} required")
        return

    c_df = pd.read_csv(cong_path); c_df["workload"] = "Congress"
    k_df = pd.read_csv(court_path); k_df["workload"] = "Court"
    df = pd.concat([c_df, k_df], ignore_index=True)

    fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4.5), sharey=True)
    for ax, wl in zip([ax1, ax2], ["Congress", "Court"]):
        sub_all = df[df["workload"] == wl]
        for policy in sorted(sub_all["policy"].unique()):
            sub = sub_all[sub_all["policy"] == policy].sort_values("alpha")
            color = POLICY_COLORS.get(policy, "gray")
            marker = POLICY_MARKERS.get(policy, "x")
            ax.plot(sub["alpha"], sub["miss_ratio"],
                    marker=marker, markersize=5, label=policy,
                    color=color, linewidth=1.5)
        ax.set_xlabel("Zipf Alpha")
        ax.set_title(wl)
        ax.set_ylim(bottom=0)
    ax1.set_ylabel("Miss Ratio (1% cache)")
    ax2.legend(bbox_to_anchor=(1.02, 1), loc="upper left", title="small_frac")
    fig.suptitle("S3-FIFO small-queue-ratio sensitivity (D-11 / ABLA-01)")

    out = os.path.join(figures_dir, "ablation_s3fifo.pdf")
    fig.savefig(out, bbox_inches="tight")
    plt.close(fig)
    print(f"  Saved {out}")
```
