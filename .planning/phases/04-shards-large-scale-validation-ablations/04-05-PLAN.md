<objective>
Axis B Part 2 (Phase 4 D-19, Wave 2): Integrate the Plan 04-02 Doorkeeper into W-TinyLFU as a sibling variant `wtinylfu-dk`, and produce the DOOR-03 ablation figure on both workloads. This is the load-bearing integration of the entire Doorkeeper axis — Plan 04-02 validated the Bloom filter in isolation; this plan wires it into the access path of the existing W-TinyLFU policy without breaking Phase 2's WTLFU-01..05 acceptance gates.

The integration has three subtle pieces that must all be correct simultaneously:

1. **CMS callback hook (D-09):** `include/count_min_sketch.h` gains a `std::function<void()> on_age_cb_` member fired from the end of `halve_all_()`. Default empty → invocations a no-op (just an `if (on_age_cb_)` check, branch-predictable). When set by WTinyLFUCache during ctor (only when use_doorkeeper_=true), the callback calls `doorkeeper_.clear()` so the DK and CMS freshness windows stay aligned (STACK.md §Doorkeeper requirement). Phase 2's WTLFU-01 conservative-update rule + sample_size formula + 4-byte halving mask + force_age() test hook are ALL UNCHANGED. The added member + setter + 1-line callback fire are the only diffs.

2. **WTinyLFUCache ctor + access() extension (D-05/D-08/D-10):** The ctor gains a third parameter `bool use_doorkeeper = false`. When true, the ctor allocates the embedded `Doorkeeper doorkeeper_` member with `n_objects_hint` (D-06 sizing), then registers a lambda `[this]{ doorkeeper_.clear(); }` on the CMS via the new setter. The access() method gains a pre-CMS-record filter at its entry point: when use_doorkeeper_=true, check `doorkeeper_.contains(key)` first; if true, call `cms_.record(key)` as before; if false, call `doorkeeper_.add(key)` and SKIP cms_.record() (the first touch is absorbed by the DK per Einziger-Friedman §4.3). When use_doorkeeper_=false, behavior is IDENTICAL to Phase 2 — the cms_.record(key) call at line 60 fires unconditionally. The admission test (admit_candidate_to_main_, D-08a..D-08e from Phase 2) is UNCHANGED — D-10 explicitly preserves "same admission, different counting."

3. **The L-12 stats-single-source invariant:** Phase 2 grep-enforced `grep -cE "record\(\s*(true|false)" include/wtinylfu.h == 4`. The DK integration MUST NOT change this count. The pre-filter ADDS no new `record(true|false)` calls — it only wraps the existing single `cms_.record(key)` at line 60 in an if-else, where each branch either calls `cms_.record(key)` once OR calls `doorkeeper_.add(key)` (which is NOT a `record(true|false)` call). The four `record(true, size)` / `record(false, size)` calls at lines 69, 82, 101, 112 are UNTOUCHED. Verifying this grep count post-edit is the load-bearing acceptance gate.

The plan also extends `src/main.cpp::make_policy` with one new branch for `wtinylfu-dk`, both label maps with the display name `"W-TinyLFU+DK"`, the Makefile with an `ablation-doorkeeper` target running both workloads through `--alpha-sweep --policies wtinylfu,wtinylfu-dk`, and `scripts/plot_results.py` with one new POLICY_COLORS/MARKERS entry plus a `plot_ablation_doorkeeper` function rendering a 2-workload-panel figure with linestyle-distinguished baseline-vs-DK lines.

def plot_ablation_doorkeeper(figures_dir, congress_dir="results/congress",
                              court_dir="results/court"):
    """W-TinyLFU Doorkeeper ablation (D-08 / DOOR-03). 2-workload panel grid.
    Reads results/{congress,court}/ablation_doorkeeper.csv, plots W-TinyLFU vs
    W-TinyLFU+DK miss_ratio vs alpha at fixed 1% cache (D-13). Shared brown
    color; linestyle (solid vs dashed) carries the with/without-DK distinction.
    """
    cong_path = os.path.join(congress_dir, "ablation_doorkeeper.csv")
    court_path = os.path.join(court_dir, "ablation_doorkeeper.csv")
    if not (os.path.exists(cong_path) and os.path.exists(court_path)):
        print(f"  Skipping Doorkeeper ablation plot: both {cong_path} and {court_path} required")
        return

    c_df = pd.read_csv(cong_path); c_df["workload"] = "Congress"
    k_df = pd.read_csv(court_path); k_df["workload"] = "Court"
    df = pd.concat([c_df, k_df], ignore_index=True)

    fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4.5), sharey=True)
    for ax, wl in zip([ax1, ax2], ["Congress", "Court"]):
        sub_all = df[df["workload"] == wl]
        for policy in sorted(sub_all["policy"].unique()):
            sub = sub_all[sub_all["policy"] == policy].sort_values("alpha")
            color = POLICY_COLORS.get(policy, "gray")
            marker = POLICY_MARKERS.get(policy, "x")
            # D-08 visual: legacy W-TinyLFU solid, +DK dashed (same color)
            linestyle = "--" if policy.endswith("+DK") else "-"
            ax.plot(sub["alpha"], sub["miss_ratio"],
                    marker=marker, markersize=5, label=policy,
                    color=color, linewidth=1.5, linestyle=linestyle)
        ax.set_xlabel("Zipf Alpha")
        ax.set_title(wl)
        ax.set_ylim(bottom=0)
    ax1.set_ylabel("Miss Ratio (1% cache)")
    ax2.legend(bbox_to_anchor=(1.02, 1), loc="upper left")
    fig.suptitle("W-TinyLFU Doorkeeper ablation (D-08 / DOOR-03)")

    out = os.path.join(figures_dir, "ablation_doorkeeper.pdf")
    fig.savefig(out, bbox_inches="tight")
    plt.close(fig)
    print(f"  Saved {out}")
