<objective>
Axis D (Phase 4 D-19): SIEVE visited-bit ablation. The Zhang et al. (NSDI'24) SIEVE paper claims that lazy promotion (setting the per-object visited bit on every hit, then clearing it as the eviction hand sweeps) is what gives SIEVE its distinctive scan-resistance. This ablation verifies that claim by comparing SIEVE-with-promotion (default, Phase 1 behavior) vs SIEVE-without-promotion (the visited bit only flips at insertion time and the hand always evicts on first encounter — collapsing SIEVE to something close to FIFO with newer-bias).

Three coordinated changes ship in this plan (SAME shape as Plan 04-03 but for SIEVE; smaller because only ONE new variant, not three):

1. **`include/cache.h::SIEVECache` ctor + hit-path extension (D-12):** Add a default-valued `bool promote_on_hit = true` parameter and a private `promote_on_hit_` + `name_` member. The ctor sets `name_` to `"SIEVE"` when true, `"SIEVE-NoProm"` when false. The `access()` hit-path at cache.h:362 (`it->second->visited = true;`) is guarded by `if (promote_on_hit_)`. The `evict_one()` body at lines 378-404 is UNTOUCHED — the hand still reads `hand_->visited` and clears it as designed; D-12 only suppresses the SET on hit, not the READ during eviction. This means SIEVE-NoProm + insertion at line 371 with `visited=false` → every entry's visited bit is born false and never set (except by the explicit insertion default which is already false) → eviction hand encounters every entry as "visited=false" on first sweep → effectively FIFO. The miss-ratio difference between SIEVE and SIEVE-NoProm IS the value of lazy promotion.

2. **`src/main.cpp` factory + label-map extension:** ONE new make_policy branch `sieve-noprom` constructs `SIEVECache(capacity, /*promote_on_hit=*/false)`. The legacy `sieve` branch stays as `SIEVECache(capacity)` (no second arg → ctor default true → name_ = "SIEVE" preserved). Both label-map blocks at lines 184-188 and 234-238 gain ONE new `else if` branch.

3. **Makefile + plot_results.py extensions:** The `ablation-sieve` Makefile target reuses the `--alpha-sweep` path with `--policies sieve,sieve-noprom` against both workloads. The new `plot_ablation_sieve` function in plot_results.py reads both per-workload CSVs and produces a 2-panel figure (Congress | Court). POLICY_COLORS/POLICY_MARKERS gain ONE new entry for SIEVE-NoProm (purple, distinguished from legacy SIEVE via dashed linestyle in the plot — implementation detail in plot function).

**Why this is its own plan (NOT folded with 04-03):** Although the patterns are nearly identical to Axis C (S3-FIFO ablation), folding both into a single 6-task plan would exceed the 2-3 task budget per CONTEXT planning rules. They also touch disjoint regions of cache.h (S3FIFOCache vs. SIEVECache) and disjoint POLICY_COLORS/MARKERS keys, so they are genuinely parallelizable as Wave 1 plans.

**Crucial back-compat constraint (D-12 + Phase 4 L-8 equivalent):** A re-run of `make run-sweep` after this plan must produce SIEVE rows in mrc.csv that are byte-identical to pre-Phase-4 outputs. Since the default `promote_on_hit_=true` causes the access() hit path to execute the SAME line of code as before (`it->second->visited = true`), this is structurally guaranteed — the if-guard wraps an unchanged statement when the condition is true. There is no FP-rounding subtlety as in Plan 04-03 because no formula changes.

```python
def plot_ablation_sieve(figures_dir, congress_dir="results/congress",
                          court_dir="results/court"):
    """SIEVE visited-bit ablation (D-12, ABLA-02). 2-workload panel grid.
    Reads results/{congress,court}/ablation_sieve.csv, plots SIEVE vs SIEVE-NoProm
    miss_ratio vs alpha at fixed 1% cache (D-13). The two variants share a color
    (purple) so the linestyle (solid vs dashed) carries the with/without-promotion
    distinction visually.
    """
    cong_path = os.path.join(congress_dir, "ablation_sieve.csv")
    court_path = os.path.join(court_dir, "ablation_sieve.csv")
    if not (os.path.exists(cong_path) and os.path.exists(court_path)):
        print(f"  Skipping SIEVE ablation plot: both {cong_path} and {court_path} required")
        return

    c_df = pd.read_csv(cong_path); c_df["workload"] = "Congress"
    k_df = pd.read_csv(court_path); k_df["workload"] = "Court"
    df = pd.concat([c_df, k_df], ignore_index=True)

    fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4.5), sharey=True)
    for ax, wl in zip([ax1, ax2], ["Congress", "Court"]):
        sub_all = df[df["workload"] == wl]
        for policy in sorted(sub_all["policy"].unique()):
            sub = sub_all[sub_all["policy"] == policy].sort_values("alpha")
            color = POLICY_COLORS.get(policy, "gray")
            marker = POLICY_MARKERS.get(policy, "x")
            # D-12 visual: SIEVE solid, SIEVE-NoProm dashed (same color)
            linestyle = "--" if policy.endswith("NoProm") else "-"
            ax.plot(sub["alpha"], sub["miss_ratio"],
                    marker=marker, markersize=5, label=policy,
                    color=color, linewidth=1.5, linestyle=linestyle)
        ax.set_xlabel("Zipf Alpha")
        ax.set_title(wl)
        ax.set_ylim(bottom=0)
    ax1.set_ylabel("Miss Ratio (1% cache)")
    ax2.legend(bbox_to_anchor=(1.02, 1), loc="upper left")
    fig.suptitle("SIEVE visited-bit ablation (D-12 / ABLA-02)")

    out = os.path.join(figures_dir, "ablation_sieve.pdf")
    fig.savefig(out, bbox_inches="tight")
    plt.close(fig)
    print(f"  Saved {out}")
```
